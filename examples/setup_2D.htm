Connect a 2D panel with WS2812 LEDs to your Raspberry pi. It's possible to chain many panels together and create a large display.
First you need to do a normal setup with total LEDs in all connected panels.
The config_2D command will create a conversion table in the background taking the x and y location and convert it to LED position in the normal 1D string. There are 4 supported LED configurations and a 5th type which allows you to create your own conversion table in a text file.
At this moment the first pixel (0,0) must always be at the left top of the screen.

  • Panel type 0:
    Row by row and the end of each row is connected to the beginning of the next row.
  • Panel type 1:
    Row by row but even rows are connected left to right and odd rows right to left.
  • Panel type 2:
    Panels with LEDs configured as columns.
  • Panel type 3:
    Panels with LEDs configured as columns but odd panel rows are rotated 180°.
  • Panel type 4:
    Custom pixel locations defined in a text file. This text file looks like:
    x,y=index 0,0=3 1,0=4 2,0=5 3,0=10 4,0=11 5,0=6
    Here x and y is the pixel coordinates and index is the location of the LED at x,y in the 1D string. (Do not include x,y=index in the file.)

The following commands configure a 32x16 pixel display with 2 32x8 pixel displays.

setup 1,512,3 init brightness 1,32 config_2D 1,32,16,3,32,8

NOTE: You need to compile with ENABLE_2D=1 for 2D commands to work. This also requires additional libraries to be installed.
If you forgot type:

make clean make ENABLE_2D=1
